# Rubik's Cube Solver - Android 魔術方塊掃描求解器

## 1. 專題簡介
本專題開發一款基於 Android 平台的魔術方塊求解 App。使用者透過手機鏡頭依序掃描魔術方塊的六個面，系統將利用電腦視覺技術自動辨識顏色分布，並透過 **Kociemba 演算法** 在秒級時間內計算出最佳復原路徑（通常在 20 步以內）。本工具旨在幫助初學者學習復原魔術方塊，或作為複雜演算法與行動裝置整合的技術實踐。

## 2. UI 特色
為了提升使用者體驗，本版本新增了直覺式的操作介面：
*   **掃描進度指示燈**：頂部設有六顆進度圓點，即時顯示各面掃描狀態。
*   **動態提示卡片**：針對每一面提供專屬的翻轉引導與掃描基準提示。
*   **辨識信心度量表**：即時回饋影像辨識品質，當信心度過低時系統會提醒調整角度。
*   **靈活錯誤修正**：支援「回上一步」功能，若掃描出錯可隨時退回上一面修正，無需全部重來。
*   **階層化解法顯示**：清楚呈現復原公式與總步數，並針對大螢幕裝置優化佈局避免導航列遮擋。

## 3. 系統需求
*   **作業系統**：Android 8.0 (API Level 26) 或更高版本。
*   **權限要求**：相機存取權限 (Camera Permission)。
*   **架構支援**：支援 arm64-v8a, armeabi-v7a, x86_64 之行動裝置。

## 4. 硬體與軟體環境
*   **開發硬體**：PC / Laptop, Android 實體行動裝置。
*   **開發軟體**：
    *   **Android Studio**：Pandas 4 | 2025.3 
    *   **Gradle**：8.0+。
    *   **Python**：3.10 (透過 Chaquopy 整合)。
*   **主要程式語言**：Java (UI/CameraX 控制), Python (影像處理/演算法)。

## 5. 安裝方式
1.  **複製專案**：
    ```bash
    git clone https://github.com/WengJunZhe/myapp.git -b main
    ```
2.  **開啟專案**：使用 Android Studio 開啟 `myapp` 資料夾。
3.  **同步 Gradle**：點擊右上角 **Sync Project with Gradle Files** 圖示，系統會自動透過 Chaquopy 下載必要的 Python 環境與套件（NumPy, OpenCV）。
4.  **演算法配置**：確保 `app/src/main/python/kociemba/` 資料夾內已包含完整的純 Python 版演算法實作。
5.  **編譯並執行**：連結 Android 裝置，點擊 **Run 'app'** 進行安裝。

## 6. 執行方式
1.  **啟動 App**：點擊畫面的「開始掃描」按鈕。
2.  **依序掃描六面**：按照螢幕上方提示的順序（白、橘、綠、紅、藍、黃）掃描。
    *   掃描時請確保方塊對準綠色對焦框。
    *   確認辨識顏色無誤且信心度足夠後，點擊「確認」進入下一面。
    *   **修正**：若掃描不理想，可點擊「回上一步」或「重新掃描」即時修正該面資料。
3.  **獲取解法**：完成六面掃描後，系統自動計算解法。
4.  **執行復原**：
    *   **重要基準**：請將魔術方塊握持為 **「白色面朝上、橘色面對自己」** 的姿態。
    *   按照畫面顯示的公式（如 `U R2 F'`）旋轉方塊即可完成復原。

## 7. 資料集說明
本專題不涉及深度學習模型訓練，而是採用電腦視覺規則提取特徵：
*   **影像輸入**：透過 CameraX 獲取即時 YUV 影格。
*   **特徵提取**：利用 OpenCV 進行邊緣檢測（Canny）與四邊形擬合。
*   **顏色空間**：將 BGR 轉換至 **HSV 顏色空間**，並定義精準的六色門檻區間（COLOR_RANGES）進行分類。

## 8. 模型或演算法說明
*   **影像辨識流程**：
    1.  **預處理**：灰階化、高斯模糊、Canny 邊緣偵測（優化門檻值以減少閃爍）。
    2.  **透視變換**：偵測到方塊邊界後，執行 `warpPerspective` 將傾斜的方塊轉為正向 3x3 矩陣。
    3.  **顏色採樣**：對 9 宮格內各個區域進行中位數（Median）採樣，避開中心塊 Logo 造成的誤判。
*   **求解演算法**：
    *   採用 **Kociemba's Two-Phase Algorithm**。
    *   該演算法將魔術方塊狀態空間分解為兩個子群，能在極短時間內搜尋到 20 步以內的解法。
    *   **座標校正**：在 Python 端自動執行順序重新排列與旋轉校正（特別針對底面黃色進行 90 度校正）。

## 9. 測試結果
*   **解法精確度**：對於合法狀態的魔術方塊，求解成功率為 100%。
*   **報錯機制**：若掃描後顏色數量不符（非 9 個）或狀態矛盾，系統會精準提示「顏色數量不正確」或「拍攝順序錯誤」。
*   **復原步數**：平均解法長度為 18-22 步。
*   **執行速度**：影像辨識延遲 < 100ms，演算法計算時間 < 5s。
